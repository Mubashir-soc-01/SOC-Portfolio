Day 9 — Windows Event Logs (Part 2) + PowerShell Logging

Practical 1: Event ID 4720 (User Created)

Step 1: Open Computer Management
Windows Key + R → lusrmgr.msc → Enter

Step 2: Create Test User
Right click Users → New User
Username: testuser
Password: Test@123
Click Create → Close

Step 3: Open Event Viewer
Windows Key + R → eventvwr.msc → Enter

Step 4: Filter Event ID 4720
Windows Logs → Security → Filter Current Log
Event IDs: 4720 → OK

Step 5: Delete Test User
Computer Management → Users → Right click testuser → Delete


Practical 2: Event ID 1102 (Log Cleared)

Step 1: Open Event Viewer
Windows Key + R → eventvwr.msc → Enter

Step 2: Clear Security Log
Windows Logs → Security → Right click → Clear Log → Click Clear

Step 3: Filter Event ID 1102
Right click Security → Filter Current Log → Event IDs: 1102 → OK


Practical 3: PowerShell Logging — Event ID 4104

Step 1: Enable Script Block Logging (Admin PowerShell)
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging" -Name "EnableScriptBlockLogging" -Value 1

Step 2: Run a Simple Command
Write-Host "PowerShell logging test"

Step 3: Run an Encoded Command (Attacker Style)
$Command = 'write-host "malicious Command"'
$bytes = [System.Text.Encoding]::Unicode.GetBytes($Command)
$encoded = [Convert]::ToBase64String($bytes)
Write-Host $encoded

Step 4: Run the Encoded Command
powershell.exe -EncodedCommand $encoded

Step 5: Open Event Viewer
Windows Key + R → eventvwr.msc → Enter

Step 6: Go to PowerShell Logs
Applications and Services Logs → Microsoft → Windows → PowerShell → Operational

Step 7: Filter Event ID 4104
Right click Operational → Filter Current Log → Event IDs: 4104 → OK

Step 8: Find your commands
Look for:
- "Write-Host PowerShell logging test"
- Encoded string (long random characters)
- Decoded command "write-host malicious Command"
